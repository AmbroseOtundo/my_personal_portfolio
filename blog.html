Server Downtime

Incident report for Helm Associates

Issue Summary

On November 6th, 2022 from 9.00 PM to 00.00 PM EST, the company's server was down for 3 hours. 100% of users experienced a 500 internal server error caused by a network intrusion which led to the fail safe activating.

Timeline

  • 9:00 PM Intrusion is detected, outage begins
  • 9:09 PM Alert is sent out
  • 9:20 PM Error logs checked by server admins team
  • 10:30 PM Network is configured
  • 10:45 PM Firewall is updated
  • 11:04 PM Intruder is locked out, turns out it was an automated attack
  • 11:59 PM Server 100% restored and back online

Root Cause and Resolution

After a network intrusion by a hacker, an alert was sent to the reliability and server admins. They went to check the logs and found an automated attack by a bot. The firewall and network was update. The intruder was fend off. By 00.00 AM the server was up and running.

Corrective and Preventative Measures

After a discussion it has been decided we can prevent this in the future by:

  • Having a second backup server to prevent the fail safe mechanism from bringing the whol system down.
  • Have pentesters to regularly test the system against automated tests and attacks. This is only a small incident so we are doing great! Thanks for your patience and your continued confidence in us.
